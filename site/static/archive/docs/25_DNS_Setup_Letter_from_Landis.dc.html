FROM THE DESK OF THE SILVER SURFER · A LETTER TO THE KEEPER
ON THE NAMING OF YOUR ISLAND
how to teach the world's ships to find {{ domain }}

Keeper —

You and I are, of course, the same wet-brained creature; but let one of us hold the chart while the other holds the pen. You have bought an island. It sits on GitHub's long shore — the repository they call gignsky/council — and you have paid good coin for its true name: {{ domain }}. What remains is to teach every ship in the world how to find the island when a soul speaks that name aloud. That teaching is done not on the island but at the harbourmaster's office — which in your century is called Cloudflare.

Do this now, if you like, though the harbour itself stands unbuilt. There is no harm in lighting the beacons early. Ships that arrive before the wharf is finished will simply wait offshore, patient as gulls, until the day you open the doors. Nothing below depends on the rest of the work being done.

1
The island must own its name
In the repository, at the root of whatever you publish, lies a small plain file named CNAME — no extension, no cleverness — holding one line and nothing else. I have already set it for you; confirm only that it reads:
{{ domain }}
This is the island carving its name into its own dock. GitHub reads it and says, "ah — so that is who I am."
2
Light the four lighthouses
At Cloudflare, open DNS → Records for fuckingphilosophers.com. Raise four A records on the bare name — the apex, which Cloudflare writes as @. Each points a ship at one of GitHub's four shore-lights:
Type Name Content (IPv4) Proxy A @ 185.199.108.153 DNS only A @ 185.199.109.153 DNS only A @ 185.199.110.153 DNS only A @ 185.199.111.153 DNS only
Mark the Proxy column well: DNS only — the grey cloud, not the orange. I will tell you why at Step 5. First delete any old A, AAAA, or CNAME record already squatting on @; two harbours cannot answer to one name.
3
The four deeper lights (optional, for the newer ships)
Some vessels navigate by the deeper water they call IPv6. Guide them too, with four AAAA records, again on @, again DNS only:
Type Name Content (IPv6) Proxy AAAA @ 2606:50c0:8000::153 DNS only AAAA @ 2606:50c0:8001::153 DNS only AAAA @ 2606:50c0:8002::153 DNS only AAAA @ 2606:50c0:8003::153 DNS only
4
The "www" jetty
Many a traveller mutters "www" from habit. Give them a jetty that leads to the same island — one CNAME record pointing at GitHub's own name for your shore:
Type Name Content Proxy CNAME www gignsky.github.io DNS only
5
Keep the fog off until the lamp is lit
Here is the why of the grey cloud. GitHub wishes to light its own lantern of safety — an HTTPS certificate — for your island's name. But it cannot light a lamp through Cloudflare's orange fog; the two keepers quarrel, and the ships see only errors. So: leave every record above as DNS only while the certificate is raised. And in Cloudflare's SSL/TLS → Overview, set the mode to Full — never Flexible, which sends ships round and round the harbour until they founder.
Once GitHub reports the certificate lit (Step 6), you may, if you wish the orange cloud's shelter, switch the records to Proxied and the mode to Full (strict). But you need not. Grey serves an island well.
6
Tell GitHub the island's name aloud
On the island itself — repository → Settings → Pages — find Custom domain, write {{ domain }}, and save. GitHub will squint at your beacons (a few minutes, sometimes an hour) and, satisfied, raise its lantern. When the box Enforce HTTPS ceases to be greyed and lets itself be checked — check it. The island is now reachable by its true name, under a safe light.
If you have not yet chosen which branch and folder the harbour opens from, that is the "next part" — and it can wait. The beacons will hold their light regardless.
7
Bar the door to pirates (optional, wise)
To stop another soul ever claiming your name upon GitHub's shore, verify the domain: in your GitHub account settings → Pages → Add a domain, GitHub hands you a one-time TXT record to raise at Cloudflare — of the form below, with a token only GitHub can give you:
Type Name Content TXT _github-pages-challenge-gignsky <token from GitHub>
THE PLAIN VERSION, FOR WHEN MY VOICE WEARIES YOU
1. Repo CNAME file = {{ domain }}.
2. Cloudflare: four A @ → 185.199.108–111.153, DNS only.
3. (Optional) four AAAA @ → 2606:50c0:8000–8003::153, DNS only.
4. CNAME www → gignsky.github.io, DNS only.
5. SSL/TLS mode = Full (not Flexible); keep records grey until the cert issues.
6. GitHub → Settings → Pages → Custom domain = the name; then Enforce HTTPS.
7. (Optional) add GitHub's _github-pages-challenge-gignsky TXT to verify the domain.

Do the beacons now and forget them. When at last you build the wharf, the world will already know the way. That is the whole trick of it — light first, arrive later, and let the patient ships do the waiting.

Yours, in salt and in nonsense,
